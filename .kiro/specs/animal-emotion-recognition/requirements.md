# Requirements Document

## Introduction

Kaggle의 "Pets Facial Expression Dataset"(https://www.kaggle.com/datasets/anshtanwar/pets-facial-expression-dataset)을 활용하여 동물의 이미지를 입력받으면 해당 동물의 감정 상태를 한국어로 표현하는 딥러닝 기반 모바일 애플리케이션을 개발한다. 해당 데이터셋은 소규모(총 약 1,074장)이므로 데이터 증강 전략을 적극 활용한다. 사용자는 카메라 촬영 또는 갤러리 선택을 통해 동물 이미지를 제공하고, 시스템은 학습된 모델을 기반으로 감정을 분류하여 한국어 텍스트로 결과를 표시한다.

## Glossary

- **앱(App)**: 사용자가 설치하여 사용하는 모바일 애플리케이션
- **감정_분류기(Emotion_Classifier)**: 동물 이미지를 분석하여 감정 카테고리를 예측하는 딥러닝 모델
- **이미지_입력기(Image_Input)**: 사용자로부터 동물 이미지를 전달받는 인터페이스 모듈
- **결과_표시기(Result_Display)**: 분류된 감정 결과를 한국어로 사용자에게 표시하는 UI 모듈
- **반려동물_표정_데이터셋(Pets_Facial_Expression_Dataset)**: Kaggle에서 제공하는 반려동물 표정 이미지 데이터셋으로, 4개의 감정 카테고리(Angry, Happy, Sad, Other)로 분류된 이미지를 포함한다
- **전처리기(Preprocessor)**: 입력 이미지를 모델이 처리할 수 있는 형태로 변환하는 모듈
- **감정_카테고리(Emotion_Category)**: 동물의 감정 상태를 분류하는 4개의 레이블 (Angry=화남, Happy=행복, Sad=슬픔, Other=기타)
- **데이터_증강기(Data_Augmentor)**: 소규모 데이터셋의 한계를 보완하기 위해 이미지를 변환하여 학습 데이터를 확장하는 모듈

## Requirements

### 요구사항 1: 이미지 입력

**사용자 스토리:** 사용자로서, 동물 이미지를 앱에 제공할 수 있어야 하며, 이를 통해 동물의 감정 분석을 수행할 수 있다.

#### 인수 조건

1. WHEN 사용자가 카메라 촬영 버튼을 누르면, THE 이미지_입력기 SHALL 기기의 카메라를 활성화하고 촬영된 이미지를 전달받는다
2. WHEN 사용자가 갤러리 선택 버튼을 누르면, THE 이미지_입력기 SHALL 기기의 사진 갤러리를 열고 선택된 이미지를 전달받는다
3. WHEN 이미지가 전달되면, THE 전처리기 SHALL 해당 이미지를 224x224 픽셀 크기로 리사이즈하고 픽셀 값을 0.0~1.0 범위로 정규화를 수행한다
4. IF 지원되지 않는 이미지 형식이 입력되면, THEN THE 이미지_입력기 SHALL 지원 형식(JPG, PNG)을 안내하는 오류 메시지를 표시하고 이미지 입력 화면을 유지한다
5. IF 이미지 파일 크기가 10MB를 초과하면, THEN THE 이미지_입력기 SHALL 파일 크기 제한(10MB)을 안내하는 오류 메시지를 표시하고 이미지 입력 화면을 유지한다
6. IF 사용자가 카메라 또는 갤러리 접근 권한을 거부하면, THEN THE 이미지_입력기 SHALL 해당 기능 사용을 위해 권한이 필요함을 안내하는 메시지를 표시하고 기기의 권한 설정 화면으로 이동할 수 있는 옵션을 제공한다
7. IF 사용자가 카메라 촬영 또는 갤러리 선택을 취소하면, THEN THE 이미지_입력기 SHALL 이전 화면 상태를 유지하고 별도의 오류 메시지를 표시하지 않는다
8. IF 이미지 해상도가 가로 또는 세로 50픽셀 미만이면, THEN THE 이미지_입력기 SHALL 최소 해상도 미달을 안내하는 오류 메시지를 표시하고 이미지 입력 화면을 유지한다

### 요구사항 2: 감정 분류 모델

**사용자 스토리:** 사용자로서, 동물 이미지를 제공하면 정확하게 동물의 감정을 분류받고 싶다.

#### 인수 조건

1. THE 감정_분류기 SHALL 반려동물_표정_데이터셋을 활용하여 학습된 CNN 기반 딥러닝 모델을 사용한다
2. WHEN 전처리된 이미지가 입력되면, THE 감정_분류기 SHALL 해당 이미지에 대해 4개의 감정_카테고리(화남, 행복, 슬픔, 기타)를 예측하고 각 카테고리별 확률값을 소수점 첫째 자리까지의 백분율(합계 100%)로 반환한다
3. THE 감정_분류기 SHALL 4개의 감정_카테고리(Angry=화남, Happy=행복, Sad=슬픔, Other=기타)를 분류한다
4. WHEN 예측이 완료되면, THE 감정_분류기 SHALL 가장 높은 확률의 감정_카테고리와 해당 신뢰도 점수를 백분율로 반환한다
5. IF 최상위 예측 신뢰도가 50% 미만이면, THEN THE 감정_분류기 SHALL 불확실 판정 메시지와 함께 상위 3개 감정_카테고리 및 각각의 신뢰도 점수를 반환한다
6. IF 입력 이미지에서 동물이 감지되지 않으면, THEN THE 감정_분류기 SHALL 동물 미감지 오류를 반환하고 감정 분류를 수행하지 않는다
7. IF 모델 추론 중 오류가 발생하면, THEN THE 감정_분류기 SHALL 추론 실패를 나타내는 오류를 반환하고 입력 이미지 데이터를 보존한다

### 요구사항 3: 한국어 결과 표현

**사용자 스토리:** 사용자로서, 동물의 감정 분석 결과를 이해하기 쉬운 한국어 문장으로 받고 싶다.

#### 인수 조건

1. WHEN 감정 분류 결과가 반환되면, THE 결과_표시기 SHALL 해당 감정을 "[동물명]이(가) [감정 표현] 것 같아요"의 문장 구조를 따르는 한국어 문장으로 변환하여 표시한다
2. THE 결과_표시기 SHALL 각 감정_카테고리(화남, 행복, 슬픔, 기타)별로 미리 정의된 한국어 표현 템플릿을 사용하며, 각 카테고리에 대해 최소 3개의 서로 다른 표현 변형 중 무작위로 하나를 선택하여 표시한다
3. WHEN 결과가 표시되면, THE 결과_표시기 SHALL 감정_카테고리에 매핑된 이모지 1개와 함께 신뢰도 점수를 정수 백분율(소수점 없이, 예: 87%)로 표시한다
4. WHEN 신뢰도가 50% 미만인 결과가 표시되면, THE 결과_표시기 SHALL 상위 3개 감정 예측 결과를 각 항목마다 감정 이모지, 감정_카테고리명, 신뢰도 백분율을 포함한 목록 형태로 함께 표시한다
5. IF 감정_분류기로부터 결과를 받지 못하거나 오류가 발생하면, THEN THE 결과_표시기 SHALL 분석 실패를 나타내는 오류 메시지를 표시하고 재시도 버튼을 제공한다

### 요구사항 4: 모바일 앱 UI/UX

**사용자 스토리:** 사용자로서, 직관적이고 사용하기 편리한 앱 인터페이스를 통해 동물 감정 분석 기능을 이용하고 싶다.

#### 인수 조건

1. THE 앱 SHALL 메인 화면에 카메라 촬영 버튼과 갤러리 선택 버튼을 제공한다
2. WHILE 감정 분류가 진행 중인 동안, THE 앱 SHALL 로딩 인디케이터를 표시하여 처리 상태를 안내한다
3. WHEN 분석이 완료되면, THE 앱 SHALL 입력 이미지, 예측된 감정_카테고리 이름, 신뢰도 백분율, 감정 이모지, 한국어 결과 문장을 하나의 결과 화면에 함께 표시한다
4. THE 앱 SHALL 분석 이력을 최근 20건까지 저장하며, 각 이력 항목에는 입력 이미지 썸네일, 예측된 감정_카테고리, 신뢰도 백분율, 분석 일시를 포함한다
5. WHEN 저장된 이력이 20건을 초과하면, THE 앱 SHALL 가장 오래된 이력을 자동으로 삭제하고 새 이력을 저장한다
6. THE 앱 SHALL Android 8.0(API 26) 이상 및 iOS 14.0 이상의 플랫폼을 지원한다
7. IF 감정 분류 처리 중 오류가 발생하면, THEN THE 앱 SHALL 로딩 인디케이터를 제거하고 오류가 발생했음을 나타내는 메시지를 표시하며 메인 화면으로 복귀할 수 있는 버튼을 제공한다

### 요구사항 5: 데이터셋 관리 및 데이터 증강

**사용자 스토리:** 개발자로서, Kaggle 반려동물 표정 데이터셋의 소규모 특성을 보완하여 모델을 효과적으로 학습시킬 수 있어야 한다.

#### 인수 조건

1. THE 데이터_증강기 SHALL 반려동물_표정_데이터셋의 Master Folder 내 사전 분할된 구조(train: 1,000장, valid: 36장, test: 38장)를 기본 데이터 분할로 활용한다
2. THE 데이터_증강기 SHALL 학습 데이터에 대해 회전(최대 ±30도), 수평 반전, 밝기 조절(±20%), 색상 변환(채도 ±20%)을 포함하는 증강 기법을 적용하여 학습 데이터를 최소 4배 이상 확장한다
3. THE 데이터_증강기 SHALL 검증 및 테스트 데이터에는 증강을 적용하지 않고 원본 이미지만 사용한다
4. WHEN 데이터셋 내 기존 증강 이미지(aug- 접두사 파일)가 포함되어 있으면, THE 데이터_증강기 SHALL 해당 이미지를 원본과 함께 학습 데이터로 활용하되, 추가 증강 시 원본 이미지만을 증강 대상으로 사용한다
5. THE 데이터_증강기 SHALL 4개 감정_카테고리(Angry, Happy, Sad, Other) 간 학습 데이터 수가 균등하도록 클래스 균형을 유지한다
6. IF 특정 카테고리의 학습 데이터가 다른 카테고리 대비 20% 이상 적으면, THEN THE 데이터_증강기 SHALL 해당 카테고리에 추가 증강을 적용하여 클래스 간 균형을 맞춘다

### 요구사항 6: 모델 학습 및 관리

**사용자 스토리:** 개발자로서, 증강된 데이터셋으로 모델을 학습하고 모바일 배포용으로 관리할 수 있어야 한다.

#### 인수 조건

1. THE 감정_분류기 SHALL 증강된 학습 데이터와 원본 검증 데이터를 활용하여 CNN 기반 모델을 학습한다
2. WHEN 모델 학습이 완료되면, THE 감정_분류기 SHALL 테스트 데이터셋(38장)에서 전체 정확도 최소 70% 이상을 달성하며, 각 감정_카테고리별 정확도는 최소 60% 이상을 달성한다
3. THE 감정_분류기 SHALL 학습된 모델을 모바일 배포용으로 TensorFlow Lite 또는 ONNX 형식으로 변환한다
4. WHEN 모델이 변환되면, THE 감정_분류기 SHALL 변환 전후 정확도 차이가 5% 이내임을 검증한다
5. IF 변환 후 정확도 차이가 5%를 초과하면, THEN THE 감정_분류기 SHALL 변환 실패를 보고하고 양자화 수준을 조정하여 재변환을 시도한다
6. THE 감정_분류기 SHALL 모델 파일 크기를 50MB 이하로 유지하여 모바일 환경에서의 배포를 지원한다
7. WHEN 검증 손실(validation loss)이 5 에포크 연속 감소하지 않으면, THE 감정_분류기 SHALL 조기 종료(early stopping)를 적용하여 과적합을 방지한다

### 요구사항 7: 성능 및 응답 시간

**사용자 스토리:** 사용자로서, 이미지를 입력한 후 빠르게 감정 분석 결과를 받고 싶다.

#### 인수 조건

1. WHEN 이미지가 입력되면, THE 앱 SHALL RAM 4GB 이상의 기기에서 10MB 이하 이미지에 대해 전처리부터 결과 표시까지 전체 과정을 3초 이내에 완료한다
2. THE 앱 SHALL 네트워크 연결 없이도 감정 분류 기능을 수행할 수 있도록 모델을 로컬에 탑재한다
3. WHILE 앱이 포그라운드에서 감정 분석을 수행 중일 때, THE 앱 SHALL 200MB 이하의 메모리를 사용한다
4. IF 기기의 가용 메모리가 50MB 이하로 감소하면, THEN THE 앱 SHALL 가장 오래된 분석 이력 캐시부터 순차적으로 삭제하고 사용자에게 캐시가 정리되었음을 알리는 알림을 표시한다
5. IF 캐시 정리 후에도 가용 메모리가 50MB 이하인 상태가 지속되면, THEN THE 앱 SHALL 감정 분석 요청을 거부하고 메모리 부족으로 분석을 수행할 수 없음을 나타내는 메시지를 표시한다
6. WHEN 앱이 최초 실행되면, THE 앱 SHALL 5초 이내에 메인 화면을 표시하고 감정 분석 기능을 사용 가능한 상태로 전환한다
